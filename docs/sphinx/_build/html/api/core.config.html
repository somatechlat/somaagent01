class admin.core.config.ExternalServiceConfig(*args, **kwargs)[source]

Bases: BaseModel

External service configuration with validation.

The original strict schema required somabrain_base_url and opa_url to be non‑empty strings. Many test environments (including the CI used for this kata) do not provide those environment variables, leading to a ValidationError during the early configuration load. To keep the configuration system robust while still encouraging proper values in production, the fields are now optional. Validation only runs when a value is supplied.

validate_opa_url()

Validate OPA URL when provided.

None is permitted for the same reason as somabrain_base_url.

validate_somabrain_url()

Validate SomaBrain URL when provided.

None is allowed for test environments; otherwise the value must be a proper HTTP(S) URL.

admin.core.config.load_config()[source]

Load and validate the full configuration.

The precedence order is:

  1. Environment variables (handled by Config via env_prefix='SA01_').

  2. config.yaml or config.json located at the repository root.
